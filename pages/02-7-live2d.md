# 02-7. 라투디

![라투디 패널](../assets/ui/panel-live2d.png)

Live2D 모델·의상·표정과 TTS 립싱크를 설정합니다. 프로그램을 실행하면 캐릭터 창과 함께 동기화됩니다.

OBS에 캐릭터 창을 넣을 때 배경 처리(그린 스크린 vs 투명 창)는 우측 **실행 제어**의 [크로마키 ON/OFF](https://wikidocs.net/372534)로 바꿉니다.

## Live2D 모델

**모델 가져오기 (ZIP)…** — Live2D 번들 `.zip`을 업로드해 라이브러리에 추가합니다.

**저장된 모델 라이브러리** — 캐릭터 폴더별로 저장된 모델 목록입니다. 항목을 고른 뒤:

- **불러오기** — 선택한 모델을 캐릭터 화면에 적용
- **선택 삭제** · **목록 새로고침** — 라이브러리 관리
- **원본 모델로 리셋** — 배포 기본 모델로 되돌림 (빨간 버튼)

![screenshot](../assets/gdoc/img_057.png)

모델·표정·의상 변경은 **프로그램 실행 중에도** 캐릭터 화면에 곧바로 반영됩니다.

## 의상

캐릭터마다 **안경·뿔·고양이 귀** 같은 토글이 다릅니다. 체크하면 캐릭터 의상이 바뀝니다.

**셔츠** — 의상 variant(예: 다른 상의)를 고릅니다.

UI에서 바꾼 값과 AI 응답의 `<glasses_on>` 같은 outfit 태그가 **서로 동기화**됩니다. 채팅으로 AI가 안경을 켜면 UI 토글도 따라 움직입니다.

## 표정

**목록 새로고침** — 캐릭터에 로드된 Live2D expression 목록을 다시 읽습니다.

**전부 끄기** — 켜져 있던 expression을 모두 끕니다.

LLM이 `<happy>` 같은 감정 태그를 내면, [감정별 프롬프트](https://wikidocs.net/372530)와 아래 **감정→표정 매핑**을 거쳐 Live2D 표정이 연결됩니다.

## 감정→표정 매핑

**감정→표정 매핑 편집…** — 「happy → expression_01」처럼 감정 문자열과 Live2D expression 이름을 짝 지어 편집합니다.

![감정→표정 매핑 편집](../assets/ui/dialog-emotion-expression.png)

캐릭터 창이 떠 있고 모델이 로드되어 있으면 **목록 새로고침**으로 실제 expression 이름이 채워집니다. 아직 모델이 없으면 「매핑 가능한 표정이 없습니다」가 나올 수 있습니다.
